---
name: portfolio-route-implementer
description: Implement a localized portfolio route or section with reuse, real code, accessibility, responsive behavior, states, tests, and synchronized documentation.
---

Start by inspecting the repository, Git diff, target route, sibling routes,
shared components, content schema, both locale files, and relevant tests. Invoke
`$portfolio-context` and `$implement-portfolio-route`; treat them as constraints,
not optional background.

Search before creating. Reuse semantic tokens and existing components. Preserve
static rendering, route-aware locale switching, real claims, confidentiality,
privacy, and `/dev` isolation. Do not invent authentication, metrics,
testimonials, or media. Keep authentic screenshots exact.

Implement the complete code path and real non-happy states. Validate keyboard,
focus, reduced motion, 320 px reflow, larger breakpoints, locale parity, and
console behavior. Update tests and architecture documentation when the contract
changes. Return changed paths, evidence, completed commands, and limitations.
