# Produce a generative portfolio asset

Use `$generative-media-production` for `$ARGUMENTS`.

1. Inspect the target route, content, authentic source media, current component,
   and `docs/generative-media.md` opportunity classification.
2. Write or confirm the production brief, responsive behavior, text-safe area,
   semantics, and performance budget before generation.
3. Perform the mandatory current-market comparison in
   `docs/generative-media.md` before recommending or using a provider. Inspect
   its account, balance, model constraints, and complete project cost. Never
   purchase, upgrade, start auto-renewal, or accept a card-backed trial without
   explicit owner approval.
4. Generate distinct directions with the selected documented provider, compare
   them in context, reject defects, and refine only the strongest candidate.
5. Export responsive optimized media, preserve authentic UI exactly, and
   integrate through existing components and tokens.
6. Update the manifest and provenance, remove rejected local files, and run the
   affected release, responsive, accessibility, and media-integrity checks.

Do not use an unresearched provider, create placeholder assets, fabricate UI or
facts, or report a generation or validation as successful without evidence.
